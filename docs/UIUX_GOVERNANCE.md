# ESG GO | UIUX 防崩壞治理規範 v1.0 (Governance Norm)

## 1. 核心治理原則 (Core Principles)
1. **一致性優先於局部炫技**: 嚴禁頁面層級私刻樣式。
2. **可理解性優先於裝飾性**: 3-5 秒內必須能理解頁面用途。
3. **主任務優先於次要資訊**: 每一頁必須定義單一核心任務。
4. **模板化優先於自由拼接**: 必須優先套用 Dashboard, List, Detail, Form, Report 模板。
5. **狀態完整性優先於靜態美觀**: 必須補齊 hover, focus, disabled, loading, error, empty 六態。
6. **工程可實作性**: 必須對應全端雙向 TypeScript 型別。

## 2. 頁面模板驗收清單 (Acceptance Checklist)

### 2.1 Dashboard 模板
- [ ] 首屏是否能立即看出系統狀態？
- [ ] 主指標是否唯一且突出？
- [ ] 警示資訊是否具備最高視覺權重？

### 2.2 List 模板
- [ ] 辨識欄位是否固定在前段？
- [ ] 狀態欄位是否易於掃描？
- [ ] 篩選器是否過度壓縮內容區？

### 2.3 Detail 模板
- [ ] 使用者是否一進頁面就知道對象是誰？
- [ ] 核心狀態是否固定在頁首？
- [ ] 附件與審計歷程是否有固定區域？

### 2.4 Form 模板
- [ ] 必填欄位標示是否明確？
- [ ] 欄位順序是否符合填寫邏輯？
- [ ] 驗證訊息是否貼近欄位？

### 2.5 Report 模板
- [ ] 是否有明確版本與日期標示？
- [ ] 指標與文字解釋是否互相對應？
- [ ] 匯出 PDF 格式是否維持秩序？

## 3. 元件驗收標準
- [ ] **觸控目標**: 所有按鈕 min-height 為 44px。
- [ ] **Design Tokens**: 100% 使用 CSS 變數，零硬編碼顏色。
- [ ] **5T 色彩**: T1藍/T2綠/T3金/T4紅/T5紫語意一致。
